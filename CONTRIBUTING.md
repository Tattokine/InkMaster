# Contributing

Takk for at du vil bidra! Her er noen enkle retningslinjer for å få endringene dine inn i prosjektet.

1. Åpne en issue først hvis du planlegger større endringer eller nye funksjoner.
2. Lag en feature branch fra `main`:

```bash
git checkout -b feature/min-beskrivelse
```

3. Følg eksisterende kodingstil (JavaScript-stil, linters osv.).
4. Skriv klare commit-meldinger: `type(scope): kort beskrivelse` (f.eks. `fix(api): fiks /health response`).
5. Lag en PR med beskrivelse av hva som endres og hvorfor. Link til relevante issues.
6. Legg til tester eller manual testinstruksjoner hvis relevant.

Code review

- PR-er vil bli vurdert og merket med kommentarer ved behov.
- Når endringene er godkjent kan PR-en merges til `main`.

Takk!
